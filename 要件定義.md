# アニメキャラクイズゲーム 設計書

## 1. 概要

複数人でリアルタイムに楽しむアニメキャラ当てクイズゲーム。出題者と回答者に役割が分かれ、回答は口頭で行う。出題者がキャラ画像・正解情報を確認しながら進行を操作し、回答者には画像のみが表示される。

---

## 2. システム構成

### 技術スタック（推奨）

| レイヤー | 技術 | 理由 |
|---|---|---|
| フロントエンド | React + TypeScript | 状態管理・型安全性 |
| リアルタイム通信 | Socket.io（WebSocket） | 出題者↔回答者リアルタイム同期 |
| バックエンド | Node.js + Express | Socket.ioとの相性 |
| データ管理 | JSON ファイル or SQLite | 軽量・開発コスト低 |
| スタイリング | Tailwind CSS | 迅速なUI構築 |

### 通信フロー

```
出題者ブラウザ ←→ WebSocket ←→ サーバー（ルーム管理）←→ WebSocket ←→ 回答者ブラウザ(s)
```

## 3. 画面設計

### 3-1. トップ画面（共通）

- ゲームタイトル
- 「出題者として参加」ボタン
- 「回答者として参加」ボタン
- ルームコード入力フィールド（回答者用）

**フロー：**
1. 出題者がルーム作成 → 4桁のルームコード発行
2. 回答者がルームコード入力 → 参加

---

### 3-2. 出題者画面

```
┌──────────────────────────────────────┐
│  ルーム: XXXX      問題 3/20         │
├───────────────────┬──────────────────┤
│                   │ ✅ 正解情報       │
│   キャラ画像      │ キャラ名: ○○○○  │
│  （大・中央）     │ 作品名: △△△     │
│                   │ 補足メモ: ...    │
├───────────────────┴──────────────────┤
│  [⏭ 次の問題]  [✓ 正解]  [✗ 不正解] │
├──────────────────────────────────────┤
│  スコアボード                        │
│  田中: 5点  鈴木: 3点  山田: 4点     │
└──────────────────────────────────────┘
```

**表示要素：**
- キャラクター画像（画面左半分、大きく）
- 正解情報パネル（画面右半分）
  - キャラクター名
  - 作品名（アニメタイトル）
  - 補足メモ（声優名、あだ名など任意）
- 操作ボタン
  - 「次の問題」：次のキャラに切り替え（回答者画面も同期）
  - 「正解」：得点加算ダイアログを開く（誰が正解したか選択）
  - 「不正解」：スキップ
- リアルタイムスコアボード

### 3-3. 回答者画面

│  問題 3/20              ⏱ 00:15      │

- 結果表示オーバーレイ（正解時: 緑・✓アニメ / 不正解時: 赤・✗アニメ）

### 3-4. 結果画面（ゲーム終了後）

- 最終スコアランキング
- 正答率（問題ごとの正解者数）
- 「もう一度」「ホームに戻る」ボタン

## 4. ゲームフロー

```
[出題者] ルーム作成
    ↓
[回答者] ルームコードで参加
    ↓
[出題者] ゲーム設定（問題数・制限時間・難易度）
    ↓
[出題者] 「ゲーム開始」ボタン
    ↓ ← ここから同期
[全員]   問題表示（キャラクター画像）
    ↓
[回答者] 口頭で回答（アプリ操作なし）
    ↓
[出題者] 正解 or 不正解 を判定してボタン押下
    ↓
[出題者] 正解の場合 → 誰が正解したか選択 → 得点加算
    ↓
[出題者] 「次の問題」で次へ
    ↓
全問終了
    ↓
[全員]  結果画面
```

## 5. データモデル

### キャラクターデータ（JSON）

```json
{
  "id": "char_001",
  "name": "ナルト",
  "anime": "NARUTO -ナルト-",
  "imageUrl": "/images/char_001.jpg",
  "difficulty": 1,
  "tags": ["少年", "忍者", "2000年代"],
  "memo": "声優：竹内順子"
}
```

### ルーム状態（サーバーメモリ）

```typescript
type Room = {
  id: string;               // ルームコード
  quizmaster: string;       // 出題者のsocket ID
  players: Player[];        // 参加者リスト
  questions: Character[];   // 出題するキャラ一覧（シャッフル済み）
  currentIndex: number;     // 現在の問題番号
  scores: Record<string, number>; // プレイヤーIDごとのスコア
  status: "waiting" | "playing" | "result";
  settings: {
    totalQuestions: number;
    timeLimit: number;      // 秒（0=制限なし）
    difficulty: "all" | "easy" | "hard";
  };
};
```

## 6. WebSocket イベント設計

| イベント名 | 送信元 | 内容 |
|---|---|---|
| `create_room` | 出題者 | ルーム作成要求 |
| `join_room` | 回答者 | ルームコードで参加 |
| `start_game` | 出題者 | ゲーム開始（設定を含む） |
| `next_question` | 出題者 | 次の問題へ（画像URLを送信） |
| `judge_correct` | 出題者 | 正解判定 + 得点者ID |
| `judge_wrong` | 出題者 | 不正解・スキップ |
| `game_end` | サーバー | 全問終了通知 |
| `sync_state` | サーバー | 参加者参加時の状態同期 |

## 7. キャラクターデータの準備方針

### A. 手動追加方式（シンプル）
- 管理画面を設けず、JSONファイルに直接追加
- 画像は `/public/images/` に配置
- Claude Codeで初期データセット（代表的なアニメ50〜100キャラ）を生成

### B. 管理画面方式（拡張性高）
- 出題者がブラウザ上でキャラ追加・編集
- 画像アップロード機能付き
- タグ・難易度管理

→ **初期実装はA、後からBに拡張する方針を推奨**

## 9. UIデザイン方針

- **テーマ**: アニメ・ゲーム感のある活発なデザイン（ダーク系 or カラフル）
- **キャラ画像**: できるだけ大きく表示（回答者は画像で勝負）
- **出題者画面**: 情報密度高め・操作しやすいボタン配置
- **アニメーション**: 正解時に派手な演出（紙吹雪・フラッシュ）、不正解時は控えめに
- **レスポンシブ**: タブレット・スマホでも動作（TV接続も考慮し大画面対応）

## 10. 今後の拡張案

| 機能 | 概要 |
|---|---|
| ブザー機能 | 回答者がボタンを押して早押しを判定 |
| ヒント機能 | 数秒後にヒント（作品名のみ表示など）を出す |
| カスタムデッキ | 出題者がキャラセットを作成・保存 |
| 観戦モード | QRコードでスコアをスマホ表示 |
| 音声効果 | 正解・不正解・タイムアップのSE |
| オンライン対応 | 遠隔プレイのためのホスティング |
